## Build process

The Antigravity port is **not hand-authored** — it is assembled from `plugin/` via `build.sh`.

```sh
bash plugin-antigravity/build.sh
```

Re-run after any `plugin/` update to propagate changes. The only hand-authored files in this directory are:

- `.agent-plugin/plugin.json` — manifest (updated manually per release)
- `hooks/hooks.json` — hook wiring (updated manually if hooks change)
- `.mcp.json` — MCP server list (updated manually per `.mcp.json` changes in `plugin/`)
- `PORTING.md` — this file
- `build.sh` — the assembly script

Everything else (`bin/`, `skills/`, `template/`, docs) is generated by `build.sh`.

## Retired hooks (not in `hooks/hooks.json`)

| Hook | Canonical script | Reason retired |
|---|---|---|
| PreCompact | `pre-compact-check.sh` | Antigravity sessions use a persistent sandbox — no context compaction event fires |
| PostCompact | `post-compact-check.sh` | Same reason |

The two scripts are **not copied to `bin/`** by `build.sh`. The `hooks/hooks.json` file does not reference them.

Mitigation: Pre-compact capture logic ("session end" snapshot) can be triggered manually via `/snapshot` or `/wrapup` before ending a long session. The `intake/pre-compact-captures/` folder in the knowledge template is preserved for manual use.

## MCP notes

The Antigravity port ships the same 12-server `.mcp.json` as `plugin/.mcp.json` with one key 
unchanged: `"google docs"` (with space, not underscore). This is the corrected form from v2.18.1 — the underscore form (`google_docs`) silently fails Cowork’s directory-entry validator (ADR-013, v2.18.1 release note).

All 5 MCP-consuming skills (v2.18.0+) ship at full strength in this port:

| Skill | `~~category` | Read/Write | Status |
|---|---|---|---|
| `/clip-thread` | `~~chat` OR `~~email` | Read | Full — Antigravity supports MCP natively |
| `/extract-doc` | `~~docs` | Read | Full |
| `/meeting-notes` | `~~docs` (paste fallback) | Read | Full |
| `/sync-decisions` | `~~docs` | Write | Full — ADR-016 Rule 22 advisory preamble preserved verbatim |
| `/digest` | All 4 categories | Read | Full composite rollup |
